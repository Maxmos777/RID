Dev-team — Escopo de implementação

ring:write-plan
langflow_client.py
  • Nova função get_tenant_service_token(schema)
  • Reutiliza tenant_service_username() e _derive_tenant_service_password() de langflow_workspace.py — sem duplicação
  • Login como serv user → JWT + API key
  • Não cria o utilizador (responsabilidade do provisioning)
langflow_auth.py
  • Adicionar TenantSchema dependency (já existe em deps.py)
  • Verificar TenantMembership do utilizador no tenant → 403 se não membro
  • Buscar Customer → verificar langflow_workspace_id → 409 se nulo
  • Incluir workspace_id na resposta
  • Deprecar langflow_user_id no response
Customer model (nova migração)
  • Novo campo: langflow_service_api_key (CharField, null=True)
  • Cache da API key do serviço para não criar nova key em cada auto-login
  • Migração: 0002_add_langflow_service_api_key
  • Alternativa: guardar em settings / Redis — trade-off a decidir
